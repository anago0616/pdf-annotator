# PDFノート - 注釈・共有アプリ

スマホのブラウザで動く PDF 管理・編集アプリ(PWA)です。インストール不要で、ホーム画面に追加すればアプリとして使えます。

## できること

- **ログイン認証** — ユーザー名 + パスワードでアカウント登録/ログイン。パスワードは scrypt でハッシュ化して保存し、API・リアルタイム同期ともログイン必須。セッションはサーバー再起動後も維持されます
- **PDF の読み込み・管理** — 「＋」ボタンで PDF を追加(**複数選択で一括取り込み可**)。端末内(IndexedDB)に保存され、一覧から開く/名前変更/削除ができます
- **タイトルごとの管理** — 取り込み時に「会議資料」などのタイトル(グループ)を付けられ、ホーム画面はタイトルごとに折りたたみ可能なセクション表示。📁 ボタンで後から別タイトルへ移動できます。共有コードで参加した文書は自動的に「共有」タイトルに入ります
- **ピンチで拡大縮小** — 2本指を広げると拡大(ピンチアウト)、つまむと縮小(ピンチイン)。ピンチ中心を基点にズームし、2本指の移動で位置調整も可能(0.5〜5倍)。＋／− ボタンでも操作できます。書き込み中に手のひらや2本目の指が触れても、描きかけの線は自動でキャンセルされ誤描画を防ぎます
- **フリーハンド書き込み** — ✏️ ペン と 🖍️ マーカー(半透明)。5色 + 太さスライダー。PDF画像の読み込みを待たずにすぐ書き始められます
- **テキスト入力** — 🅣 ツールでページをタップして文字を入力。既存テキストをタップすると再編集・削除
- **消しゴム / 元に戻す** — 🧽 でなぞって削除、↩️ で1操作ずつ取り消し
- **自動保存** — 書き込みは自動で端末に保存されます
- **共有** — 📤 ボタンで注釈を焼き込んだ PDF を生成し、スマホの共有メニュー(LINE・メール・AirDrop 等)で複数人に送信。共有非対応のブラウザではダウンロードになります
- **リアルタイム共同編集(タイトル単位)** — 共有コードは**タイトル(仕事)ごとに1つ**。ホーム画面のタイトル見出しの「🔗 共有」ボタン(または編集画面の 🔗)でコードを発行すると、そのタイトル内の全文書がまとめて共有されます。参加者は「🔗 コードで参加」にコード1つを入力するだけで全文書を受信。共有中のタイトルに後から取り込んだ PDF も全員に自動配信されます。書き込み(ペン・マーカー・テキスト・消しゴム・元に戻す)は文書ごとに即座に同期、ヘッダーに接続人数を表示、切断時は自動再接続

## 使い方(ローカルで試す)

```
cd pdf-annotator
npm install        # 初回のみ (ws パッケージ)
node server.js
```

ブラウザで http://localhost:8741 を開きます。

同じ Wi-Fi のスマホから使う場合は、PC の IPアドレスで `http://<PCのIP>:8741` を開きます
(`ipconfig` で IPv4 アドレスを確認。Windows ファイアウォールで Node の受信許可が必要な場合があります)。

### 共同編集の流れ

1. 各自アプリを開いてアカウント登録(初回のみ)・ログイン
2. Aさん: ホーム画面のタイトル見出しの「🔗 共有」をタップ → そのタイトル(仕事)に1つの共有コード(例 `4AJI7Y`)が発行される
3. Bさん: ホーム画面の「🔗 コードで参加」→ コードを入力 → タイトル内の全文書がまとめて届く
4. 以降、お互いの書き込みがリアルタイムに反映される(ヘッダーの「● 共有中 N人」に参加者数、長押し/ホバーで参加者名を表示)
5. Aさんがそのタイトルに新しい PDF を取り込むと、Bさんにも自動で配信される
6. 文書はサーバーの `data/` フォルダに保存され、後から参加しても全注釈が復元されます

通信が不安定で一時切断されても、切断中の書き込みは保持され、再接続時に自動で送信されます(ACK方式)。

`node test-client.js <コード> <トークン>` で同期動作を確認できる検証用スクリプトも入っています(トークンは `/api/login` で取得)。

## スマホアプリとして使う(PWA)

このアプリは PWA 対応です。HTTPS の URL をスマホで開いてホーム画面に追加すると、**全画面のネイティブアプリのように起動**します。アプリ本体は端末にキャッシュされるため、電波の悪い場所やオフラインでも起動して書き込みできます(共同編集の同期は再接続時に自動送信)。

- **iPhone (Safari)**: 共有ボタン(□↑)→「ホーム画面に追加」
- **Android (Chrome)**: メニュー(︙)→「アプリをインストール」(または「ホーム画面に追加」)

### 方法1: すぐ試す — ワンクリック公開(Cloudflare Tunnel)

[スマホで使う.bat](スマホで使う.bat) をダブルクリックすると、サーバー起動と同時に `https://～.trycloudflare.com` の一時URLが発行されます。それをスマホで開くだけです。

- アカウント登録不要・無料
- URL は起動のたびに変わり、ウィンドウを閉じる(またはPCをスリープする)と切れます
- 初回のみ `tools\cloudflared.exe` が必要です(セットアップ済み)

### 方法2: 常用する — クラウドにデプロイ(Render 無料枠)

URL を固定して24時間使えるようにするには、`pdf-annotator` フォルダを GitHub にプッシュして [Render](https://render.com) に接続します(`render.yaml` 設定済み、ビルド不要):

1. GitHub にこのフォルダをリポジトリとしてプッシュ
2. Render で「New +」→「Blueprint」→ リポジトリを選択 → デプロイ
3. 発行された `https://～.onrender.com` をスマホで開いてホーム画面に追加

注意: Render 無料枠はディスクが一時的なため、再デプロイ時にサーバー上の共有ルーム・アカウントが消えます(各端末内の文書は残ります)。確実に残すには有料の Persistent Disk を `data/` にマウントしてください。

## 構成

| ファイル | 役割 |
|---|---|
| `index.html` | 画面構成(ホーム / 編集 / テキスト入力 / 共有コードダイアログ) |
| `app.js` | ロジック(IndexedDB 保存、pdf.js 表示、描画、pdf-lib 書き出し、WebSocket 同期) |
| `style.css` | スマホ向け UI スタイル |
| `server.js` | 共同編集サーバー(静的配信 + 認証 API + 共有ルーム API + WebSocket 中継) |
| `test-client.js` | 同期検証用スクリプト |
| `manifest.json` / `sw.js` | PWA(ホーム画面追加)対応 |

ライブラリは CDN 読み込み(pdf.js 3.11 / pdf-lib 1.17)のため、初回表示にはネット接続が必要です。

## 制限・今後の拡張

- 共有コードを知っている**ログイン済みユーザー**は誰でも参加できます。特定メンバーだけに限定したい場合は、ルームへの招待制(参加許可リスト)の追加が必要です
- ログイン状態は端末のブラウザ単位です。同じ端末のアプリ内文書一覧(IndexedDB)はユーザーを切り替えても共有されます
- 通信は HTTP/WS のため、インターネット公開時は必ず HTTPS のホスティング(Render 等)を使ってください(パスワード保護のため)
- 同時編集の競合は「ストローク追加はマージ、消しゴム・テキスト編集・元に戻すはページ単位の後勝ち」で解決します。同じページを同時に消し合うような操作は後の操作が優先されます
- App Store / Google Play 配布が必要な場合は Capacitor でそのままネイティブアプリ化できます
